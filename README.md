# 5G FFR Reproducibility Showcase

This repository is a public, synthetic-data showcase for a 5G fractional frequency reuse (FFR) simulation workflow. It recreates the shape of a MATLAB-style paper validation pipeline in Python: build repeatable SNR and user-density sweeps, compare IFR3/SWF/FFR variants, verify qualitative trends, and generate paper-style figures.

The included data is synthetic/demo data generated by the code in this repository. It is not a copy of private MATLAB source files or unpublished raw experiment outputs.

## Scope

- Topic: 5G cellular / heterogeneous network interference mitigation with fractional frequency reuse.
- Methods: `IFR3`, `SWF`, `FFR+IFR3`, and `FFR+SWF`.
- Metrics: capacity, spectral efficiency, edge capacity, edge SINR, and an interference index.
- Reproducibility style: deterministic Python equivalent of a MATLAB-style alignment workflow.
- Paper status note: public metadata can be updated later. If a DOI is shown in a fork or release, treat it as external bibliographic metadata, not as proof that these synthetic outputs are original paper data.

## Quick Start

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -e ".[test]"
python scripts/run_paper_alignment.py
python scripts/verify_paper_trends.py
python scripts/generate_figures.py
pytest
```

Generated CSV, JSON, and PNG files are written under `outputs/`.

## Why Synthetic?

This repository is intended for public portfolio review. It demonstrates the simulation design, reproducibility discipline, and communication-network reasoning without shipping private paths, original MATLAB code, sensitive account details, or raw unpublished materials.

## Expected Trend Checks

`verify_paper_trends.py` checks that:

- capacity increases with SNR for each method at the median user density;
- FFR variants improve edge-user SINR against their non-FFR counterparts;
- interference increases as user density grows;
- SWF improves total capacity against the corresponding IFR3-style baseline.
